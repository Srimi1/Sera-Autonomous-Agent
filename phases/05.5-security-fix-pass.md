## Deliverables

### Critical fixes
- **C1 — Approval threshold off-by-one.** `cli/main.py` builds threshold from config without `+1`; gate at `tier >= floor`. Rename config key `approval_required_above` → `approval_required_at_or_above` with back-compat alias.
- **C2 — `shell_run` deny-by-default.** Tool base permission flips to `DANGEROUS`. New `SAFE_ALLOWLIST` set of regex patterns (git status/diff/log, ls, cat under workspace, pwd, python -V, node -v, head, tail, wc, find, grep) downgrades to `EXECUTE`. Explicit `EXTRA_DENY` patterns (pipe-to-shell, chmod, chown, crontab, launchctl, `> ~/.ssh`, `> /etc`) hold at `DANGEROUS` even if they'd otherwise pass the allowlist.
- **C4 + H3 — Traceback & secret sanitization.** New `sera/safety/redact.py::redact_secrets()` strips `sk-…`, `sk-ant-…`, `ghp_…`, `xoxb-…`, `Bearer …`, and value-side `OPENAI_API_KEY`/`ANTHROPIC_API_KEY`/`AWS_SECRET_ACCESS_KEY` matches. `dispatcher.execute` returns `f"{type(e).__name__}: {redact(short_msg)}"` to the LLM; full traceback goes via stdlib `logging` to `~/.sera/sera.log`. CLI error path uses the same redactor.

### High fixes
- **H1 + H2 — Tool-output fence injection.** Per-session HMAC minted on `Session.create()` (`os.urandom(16)`). `compact_session` embeds `FENCE:<hmac8>` instead of raw `FENCE`. `_build_view` validates HMAC on any assistant message starting with the fence; forged fences (no HMAC / wrong HMAC) get stripped before the LLM sees them. Tool output passes through `scrub()` + literal-FENCE strip before `session.append`.
- **H5 — Shell env isolation.** `shell_run` builds env from explicit allowlist (`PATH`, `HOME`, `LANG`, `TERM`, `USER`, `SHELL`) — never inherits `*_API_KEY` / `*_TOKEN` / `*_SECRET`.
- **H6 — Shell output cap.** 1 MB tail of stdout + 1 MB tail of stderr; larger gets prefix `…[truncated, N bytes total]`. Timeout path now `proc.wait()` after `proc.kill()`.
- **H8 — Value-side secret redaction in CLI echo.** `_shorten_args` scans VALUES (not just key names) via `redact_secrets`. Same redactor wrapped over `tool_calls` JSON before SQLite persist.

### Real bugs
- Anthropic `try/except` moved inside `async with` body so `ContextOverflow` is reachable (was dead code on `await stream_ctx`).
- `dispatcher.execute` uses `asyncio.iscoroutinefunction(handler)` to choose `await handler(...)` vs `await asyncio.to_thread(handler, ...)`; drop the no-op `run_in_executor(lambda: result)`.
- `run_turn` keeps `final_text = assistant_text` on every iteration so max-iterations cap returns the last partial answer instead of just `"[max iterations reached]"`.
- Delete dead `if classify(cmd) == Permission.DANGEROUS: pass` branch in `shell_run`.
- `cli/main.py` precomputes the env-var name before the f-string (Python 3.11 parser dislikes nested dict literal in f-string under PEP 701).
- `compressor.py` tail-orphan: when tail starts with assistant containing tool_calls whose results are in middle, either pull tools forward into tail or strip the orphaned `tool_calls` field.

### Perf quick wins
- **W1** — Per-message token cache in `tokens.py`: dict keyed by `id(msg)` with content/tool_calls hash invalidation; drops `estimate_messages` on 200 msgs from ~15 ms to <0.5 ms.
- **W2** — `Session._conn` persistent; schema run once in `create()` / `load()`; `PRAGMA journal_mode=WAL`, `synchronous=NORMAL`. Cuts `append` from ~1 ms to ~50 µs.
- **W4** — `MAX_PARTIAL_KEEP = 16` (was 32). Early-return on empty chunk.
