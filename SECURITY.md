# Security Policy

## Supported versions

The latest release on the `main` branch receives security updates. Older
versions are not maintained unless stated otherwise.

## Reporting a vulnerability

Please report security issues **privately** — do not open a public issue, as
that exposes the problem before a fix is available.

Send a report to **srijanofficial007@gmail.com** with:

- a description of the issue and its impact,
- steps to reproduce (a proof of concept if you have one),
- affected version(s) or commit,
- any suggested remediation.

You can also use GitHub's private vulnerability reporting via the **Security**
tab of this repository if it is enabled.

## What to expect

- **Acknowledgement** within 72 hours.
- An initial assessment and severity rating shortly after.
- Coordinated disclosure: we will agree on a timeline before any public
  details are shared, and credit you in the advisory unless you prefer to
  remain anonymous.

Thank you for helping keep this project and its users safe.
