# Contributing to Sera-Autonomous-Agent

Thanks for taking the time to contribute. This guide covers how to propose
changes and what to expect.

## Ways to contribute

- **Report a bug** — open an issue using the Bug Report form.
- **Request a feature** — open an issue using the Feature Request form.
- **Improve docs** — typos, clarifications, and examples are always welcome.
- **Submit code** — fix a bug or build a feature via a pull request.

```bash
git clone https://github.com/Srimi1/Sera-Autonomous-Agent.git

# install dependencies (adjust for the project's stack)
```

Run the test suite before you push:

```bash
# the project's test command
```

## Pull request process

1. Fork the repo and create a branch from `main` (`feature/...` or `fix/...`).
2. Make your change. Keep it focused — one logical change per PR.
3. Add or update tests so the change is covered.
4. Ensure the full test suite passes locally.
5. Open a pull request and fill in the template. Link any related issue.
6. A maintainer will review. Address feedback by pushing new commits to the
   same branch.

## Commit messages

Write clear, present-tense messages that explain the *why*. Reference issues
where relevant (`Fixes #123`).

## Code style

Match the surrounding code. If the project has a linter or formatter, run it
before committing.

## Reporting security issues

**Do not** open a public issue for a security vulnerability. See
[SECURITY.md](SECURITY.md) for the private reporting process.

## Code of Conduct

This project follows a [Code of Conduct](CODE_OF_CONDUCT.md). By participating,
you agree to uphold it.
